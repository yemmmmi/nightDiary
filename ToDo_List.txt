#更新日志v1.2
    使用时发现RAG、MCP服务处理流程完全串行, 这导致了响应速度太长(10-15s), 本次更新引入redis缓存，旨在减少响应时长。

    【新增功能】
    - 公开日记专栏页面（/column），访客无需登录即可浏览 is_open=true 且已发布的日记
    - 日记发布/下架接口（POST/DELETE /public/column/entries/{nid}/publish）
    - diary_entries 表新增 published_to_column、publish_time 字段，区分"公开"与"已发布到专栏"
    - 前端 ColumnPage + ColumnDiaryCard + ColumnDiaryDetail 三个组件，风格与夜记主站一致

    【Redis 缓存层】
    - 引入 redis[hiredis] 异步客户端，通过 FastAPI lifespan 管理连接池（REDIS_URL + REDIS_POOL_SIZE 可配置）
    - Cache-Aside 旁路缓存：公开日记列表和详情优先读 Redis，未命中再查 MySQL 并回填
    - 缓存穿透防护：对不存在的日记 ID 缓存空值标记（column:null:{nid}，TTL=60s），防止恶意请求穿透到数据库
    - 缓存击穿防护：热点日记缓存过期时，通过 Redis 分布式锁（SET NX EX + Lua 原子释放）确保仅一个请求回源 DB
    - 缓存雪崩防护：所有缓存 TTL 加随机偏移（列表 300±120s，详情 600±60s），避免大量 key 同时过期
    - 缓存一致性：日记更新/删除/下架时自动删除相关缓存，失败时依赖 TTL 过期保证最终一致性
    - is_open 从 true 改为 false 时自动下架并清除缓存

    【限流】
    - 基于 Redis Sorted Set 的滑动窗口 IP 限流（默认 60 次/分钟）
    - 响应头包含 X-RateLimit-Limit / Remaining / Reset
    - 超限返回 HTTP 429

    【降级容错】
    - Redis 不可用时所有功能降级为直接查 MySQL，不影响核心业务
    - 缓存删除失败仅记录日志，不阻塞主流程
