> 本文件记录当前数据库全部表结构，共 9 张 MySQL 表 + Redis 数据结构 + Chroma 向量集合。

---

| UID                | int                     | NO   | PRI | NULL              | auto_increment    |
| user_name          | varchar(50)             | NO   |     | NULL              |                   |
| email              | varchar(100)            | YES  |     | NULL              |                   |
| password_hash      | varchar(255)            | NO   |     | NULL              |                   |

| age                | int                     | YES  |     | NULL              |                   |
| phone              | varchar(20)             | YES  |     | NULL              |                   |
| address            | varchar(255)            | YES  |     | NULL              |                   |
| role               | varchar(20)             | YES  |     | user              |                   |
| create_time        | datetime                | YES  |     | CURRENT_TIMESTAMP | DEFAULT_GENERATED |
| last_time          | datetime                | YES  |     | NULL              |                   |
| long_term_profile  | text                    | YES  |     | NULL              |                   |

## 7. feedback 表（用户反馈表）

```
+----------------+-------------+------+-----+-------------------+----------------+
| Field          | Type        | Null | Key | Default           | Extra          |
+----------------+-------------+------+-----+-------------------+----------------+
| id             | int         | NO   | PRI | NULL              | auto_increment |
| user_id        | int         | NO   | MUL | NULL              |                |
| diary_nid      | int         | NO   |     | NULL              |                |
| response_style | varchar(32) | NO   |     | NULL              |                |
| feedback_type  | varchar(16) | NO   |     | NULL              |                |
| reason         | varchar(64) | YES  |     | NULL              |                |
| source         | varchar(16) | NO   |     | NULL              |                |
| signal_type    | varchar(32) | YES  |     | NULL              |                |
| created_at     | datetime    | NO   |     | CURRENT_TIMESTAMP |                |
+----------------+-------------+------+-----+-------------------+----------------+
```

**字段说明：**
- `response_style`：回应使用的风格（empathetic / practical / philosophical / humorous）
- `feedback_type`：反馈类型（positive / negative）
- `reason`：负向反馈原因（too_long / too_short / irrelevant / too_generic / lacks_suggestion）
- `source`：反馈来源（explicit = 点赞点踩 / implicit = 行为信号）
- `signal_type`：隐式信号类型（read_complete / inspired_writing / frequent_usage），显式反馈时为 NULL

## 8. style_preferences 表（风格偏好表 — Thompson Sampling 参数）

```
+------------+-------------+------+-----+-------------------+----------------+
| Field      | Type        | Null | Key | Default           | Extra          |
+------------+-------------+------+-----+-------------------+----------------+
| id         | int         | NO   | PRI | NULL              | auto_increment |
| user_id    | int         | NO   | MUL | NULL              |                |
| style      | varchar(32) | NO   |     | NULL              |                |
| alpha      | float       | NO   |     | 1.0               |                |
| beta       | float       | NO   |     | 1.0               |                |
| updated_at | datetime    | NO   |     | CURRENT_TIMESTAMP |                |
+------------+-------------+------+-----+-------------------+----------------+
```

**字段说明：**
- 每个用户每种风格一条记录（user_id + style 唯一）
- `alpha`：Beta 分布正向参数（正向反馈 +1）
- `beta`：Beta 分布负向参数（负向反馈 +1）
- 新用户初始值 alpha=1, beta=1（均匀先验）
- Thompson Sampling 从 Beta(alpha, beta) 采样选择最优风格

## 9. knowledge_entries 表（结构化知识条目表）

```
+--------------+-------------+------+-----+-------------------+----------------+
| Field        | Type        | Null | Key | Default           | Extra          |
+--------------+-------------+------+-----+-------------------+----------------+
| id           | int         | NO   | PRI | NULL              | auto_increment |
| user_id      | int         | NO   | MUL | NULL              |                |
| diary_nid    | int         | NO   |     | NULL              |                |
| entity_type  | varchar(32) | NO   |     | NULL              |                |
| entity_data  | text        | NO   |     | NULL              |                |
| extracted_at | datetime    | NO   |     | CURRENT_TIMESTAMP |                |
+--------------+-------------+------+-----+-------------------+----------------+
```

**字段说明：**
- `entity_type`：实体类型（person / event / place / topic / mood）
- `entity_data`：JSON 格式的实体数据，结构因类型而异：
  - person: `{"name": "小明", "relation": "同事"}`
  - event: `{"description": "项目截止", "date": "2024-01-15"}`
  - place: `{"name": "公司", "type": "workplace"}`
  - topic: `{"name": "工作压力", "sentiment": "negative"}`
  - mood: `{"score": -0.3, "keywords": ["焦虑", "压力"]}`
- 所有查询强制 user_id 过滤，确保数据隔离

## Redis 数据结构

```
+-------------------------------+------------+----------------------------+-----------+
| Key Pattern                   | Type       | 用途                       | TTL       |
+-------------------------------+------------+----------------------------+-----------+
| memory:episodic:{uid}         | Sorted Set | Episodic Memory 情景记忆   | 永久      |
| weather:{uid}                 | String     | 天气缓存                   | 3600s     |
| column:list:{page}:{size}     | String     | 公开专栏列表缓存           | 300±120s  |
| column:detail:{nid}           | String     | 公开专栏详情缓存           | 600±60s   |
| column:null:{nid}             | String     | 空值缓存（穿透防护）       | 60s       |
| lock:column:{nid}             | String     | 分布式锁（击穿防护）       | 5s        |
| rate_limit:{ip}               | Sorted Set | IP 限流滑动窗口            | 60s       |
+-------------------------------+------------+----------------------------+-----------+
```

**Episodic Memory 详细说明：**
- Key: `memory:episodic:{user_id}`
- Score: importance 分数（0.0-1.0）
- Member: JSON 序列化的 EpisodicEntry（event, emotion, ai_suggestion, user_feedback, timestamp, diary_nids）
- 容量限制：每用户最多 100 条，超出时驱逐 importance 最低的条目
- 写入条件：importance > 0.5
- Redis 不可用时优雅降级，不影响核心分析流程

## Chroma 向量数据库

```
+-----------------------------------+----------------------------+----------------------------+
| Collection                        | Embedding                  | 用途                       |
+-----------------------------------+----------------------------+----------------------------+
| user_{uid}_diary                  | text2vec-base-chinese(768d)| 用户日记向量（按用户隔离） |
| domain_knowledge_psychology       | text2vec-base-chinese(768d)| 心理学领域知识库（共享）   |
+-----------------------------------+----------------------------+----------------------------+
```

**用户日记集合：**
- Document ID 格式：`diary_{nid}_chunk_{chunk_index}`
- Metadata 字段：nid, uid, date, tags, chunk_index, chunk_total

**领域知识集合（85 条）：**
- Document ID 格式：`{category}_{序号}`（如 cbt_001, mindfulness_005）
- Metadata 字段：category, topic, source
- 覆盖领域：CBT(20)、正念(15)、睡眠卫生(15)、社会支持(15)、情绪调节(20)
- 只读集合，所有用户共享，通过 `scripts/init_domain_knowledge.py` 初始化
