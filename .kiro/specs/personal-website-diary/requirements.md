# 需求文档

## 简介

本项目是"夜记管理系统"，一个支持多用户的 Web 日记平台。用户可以安全地记录日记、管理标签、配置个人 AI 模型接口，系统通过 LLM 大模型（ReAct Agent）对日记内容进行深度分析并给出积极回应，帮助用户养成记录生活的习惯。

技术栈：Vue 3 + TypeScript + Tailwind CSS（前端）、Python + FastAPI（后端）、LangChain（LLM 集成）、MySQL（数据库）。

---

## 词汇表

- **系统（System）**：整个夜记管理系统，包含前端和后端。
- **日记系统（Diary_System）**：负责日记的创建、存储、查询和 AI 分析的子系统。
- **AI 分析器（AI_Analyzer）**：基于 LangChain 封装的 ReAct Agent，支持 RAG、MCP、function calling、prompt 工程、上下文记忆等技术栈。
- **日记条目（Diary_Entry）**：用户在某一天撰写的一篇日记，包含日期、正文内容、天气、AI 回应等。
- **标签（Tag）**：用于分类日记的标记，分为系统预设标签和用户自定义标签，字数不超过 15 个字。
- **模型提供商（Model_Provider）**：用户自行配置的 LLM 接口信息（base_url、model_key、model_name）。
- **用户（User）**：使用本网站的普通用户。
- **管理员（Admin）**：系统维护者，拥有更高权限。
- **数据库（Database）**：MySQL 数据库。
- **API 服务（API_Service）**：FastAPI 后端服务，提供 RESTful 接口。
- **LLM 提供商（LLM_Provider）**：可配置的大模型服务商，支持 OpenAI、DeepSeek、通义千问等。

## 需求

### 1. 系统通用需求

#### 1.1 系统架构

系统采用 BS 架构，前端使用 Vue 3 + TypeScript + Tailwind CSS，后端使用 Python + FastAPI，数据库使用 MySQL。

#### 1.2 安全性

所有密码必须经过 bcrypt 哈希加密存储，严格遵守不可逆加密原则。用户 Model 的 API Key 必须加密存储，防止泄露。

#### 1.3 认证机制

系统使用 JWT 认证，所有受保护接口需携带有效 token。

---

### 2. 用户管理模块

#### 2.1 用户注册

用户需输入用户名、邮箱、密码进行注册。系统后端对密码进行哈希加密，生成 JWT Token 返回前端。

#### 2.2 用户名唯一性

系统不允许重复的用户名或邮箱注册。

#### 2.3 用户登录

用户输入用户名/邮箱和密码，验证通过后返回 JWT Token，前端本地存储维持登录状态。

#### 2.4 密码验证

登录时系统验证密码哈希，不匹配时返回错误。

#### 2.5 JWT 认证保护

所有日记、标签、分析、模型相关接口均需携带有效 JWT Token，否则返回 401。

#### 2.6 Token 失效处理

无效或过期的 Token 应返回 401 Unauthorized。

#### 2.7 用户信息修改

用户可修改个人信息（用户名、邮箱、性别、年龄、电话、地址等），修改邮箱需先发送验证码至原邮箱验证身份。

#### 2.8 用户退出

用户退出后前端清除本地 Token，后端实时更新 `last_time` 字段。

#### 2.9 用户删除

普通用户可删除自己的账号（逻辑删除或物理删除）。管理员可封禁违规用户或强制删除违规账号。

#### 2.10 用户角色

系统支持两种角色：普通用户（User）和管理员（Admin）。

---

### 3. 日记管理模块

#### 3.1 新增日记

用户撰写日记内容，可选择多个标签（Tags）。系统将日记存入日记表，并在日记-标签关联表中建立 M:N 关系记录。

#### 3.2 日记内容非空

日记正文内容不能为空或仅由空白字符组成。

#### 3.3 修改日记

用户可修改日记正文及关联标签。系统需同步更新日记表及关联表，维护标签引用计数（UsageCount）。

#### 3.4 删除日记

支持逻辑删除或物理删除。删除时触发级联操作，更新标签引用次数，并同步删除对应分析记录。

#### 3.5 日记隐私控制

日记支持 `is_open` 字段，用户可将日记设为不公开模式以保护隐私。

#### 3.6 天气与日期

Agent 调用 MCP 服务自动获取当前日期和天气，存入日记记录。

#### 3.7 AI 回应

日记创建后可触发 AI 分析，分析结果（AI_ans）存储在日记表中。

---

### 4. 日记查询模块

#### 4.1 用户数据隔离

日记查询接口只返回当前用户的日记，不暴露其他用户数据。

#### 4.2 单篇日记访问控制

用户只能访问自己的日记，访问他人日记返回 403 或 404。

#### 4.3 日记列表排序

日记列表按时间倒序排列，支持分页。

#### 4.4 标签筛选

支持通过标签 ID 筛选日记，利用标签索引提升检索性能。

---

### 5. 标签管理模块

#### 5.1 标签种类

标签分为系统预设标签和用户自定义标签。

#### 5.2 标签字数限制

标签名称不超过 15 个字。

#### 5.3 标签唯一性

系统需保证标签名称的唯一性，防止重复数据。

#### 5.4 标签列表

展示所有标签（含系统预设与用户自定义），支持按热度（UsageCount）排序。

#### 5.5 标签维护

管理员可新增系统预设标签、修改标签属性（名称、颜色）、合并重复标签。

#### 5.6 标签引用计数

日记关联或取消关联标签时，系统自动维护标签的 UsageCount。

---

### 6. 模型管理模块

#### 6.1 模型注册

用户提供 base_url、model_key（API Key）、model_name，系统自动分配 Mod_ID。

#### 6.2 API Key 加密

model_key 必须加密存储，防止泄露。

#### 6.3 模型修改

用户可修改模型的相关信息（base_url、model_key、model_name）。

#### 6.4 模型删除

用户可删除自己的模型配置。

#### 6.5 活跃度检测

系统定时检测模型的活跃度（is_active），便于开发人员评估用户活跃度。

---

### 7. 分析管理模块

#### 7.1 新增分析

提供日记 NID，系统调用 LLM ReAct Agent 生成分析日志。生成前读取该日记关联的标签作为上下文（Few-shot），辅助 AI 进行更精准的分析。

#### 7.2 AI 用户隔离

AI 分析时读取的日记数据仅包含当前用户的日记，不包含其他用户数据。

#### 7.3 删除分析

通过 Thk_ID 或 NID 删除分析日志，用于清理无效数据或重新生成。

#### 7.4 智能防重机制

修改分析时，系统先检测日记内容是否有变化。若无变化则退回申请，防止 Token 浪费；若有变化则调用 Agent 重新生成。

#### 7.5 分析质量要求

- 输出符合用户要求风格（如语气平和/严厉）
- 输出符合事实，AI 幻觉少
- Token 消耗合理
- 输出可调节性高，代码可维护性规范

---

### 8. 报表打印模块

#### 8.1 打印日记列表

将用户查询到的日记列表（包含日期、天气、心情摘要）导出为 HTML 或 PDF 格式，支持浏览器直接打印。

#### 8.2 打印个人信息

生成包含用户基本信息（用户名、注册时间等）及统计信息（日记总数、常用心情等）的报表。

---

### 9. 在线帮助模块

#### 9.1 新手引导

用户首次进入"写日记"或"查看分析"页面时，通过高亮遮罩和气泡提示逐步演示核心流程。

#### 9.2 常见问题（FAQ）

提供静态帮助页面，分类展示高频问题，支持关键词搜索。

#### 9.3 联系客服/反馈

提供在线留言板，用户可提交反馈，管理员可在后台查看并处理。

---

### 10. 系统备份/恢复模块

#### 10.1 系统备份

管理员定期将 MySQL 数据库中的关键数据导出为 SQL 文件存储备份。

#### 10.2 系统恢复

管理员可通过导入备份的 SQL 文件将系统数据恢复到最近的正常状态。
